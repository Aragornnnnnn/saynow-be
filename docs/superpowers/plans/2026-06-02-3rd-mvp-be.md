# SayNow 3차 MVP BE Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Obsidian `3차 MVP.md` 기준으로 SayNow BE를 슬롯 기반 역할극에서 4개 고정 질문 프리톡 세션과 품질 중심 피드백 저장 흐름으로 전환한다.

**Architecture:** 기존 인증, 공통 응답, NPS, observability 틀은 유지한다. `scenario_questions`를 새 질문 원장으로 두고, 세션 턴은 원본 질문 FK와 실제 노출된 `aiQuestion` snapshot을 저장한다. AI 계약은 `next-question`, `turn-feedback`, `session-feedback` 세 갈래로 나누고, 최종 피드백 생성 시 DB 저장과 세션 완료를 확정한다.

**Tech Stack:** Java 21, Spring Boot 4.0.6, Spring MVC, Spring Data JPA, Flyway, PostgreSQL, H2 PostgreSQL mode tests.
